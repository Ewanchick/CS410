PAGE / UI STRUCTURE

MAIN BOX - centered horizontally, decent amount of top padding
	> HUD - anchored to bottom of MAIN BOX
		> ITEM BOXES - centered horizontally, spaced out
			> ITEM SPRITES - shrink to fit box

	> BACKGROUND IMAGE - stacked vertically on top of HUD, same width, no bottom padding
		> elements on top of background:
		   > item sprites - appear BEHIND monsters
		   > monster srites - appear BEHIND hands
		   > player hands - appear BEHIND text boxes
		   > text boxes - TOP element

IMPORTANT !
dynamically changing hands: 
	> click on item in HUD
	   > light up item in HUD
	   > update hand sprite with image based on value of image in HUD item box

	   > based on value of item in the box, context menu options change:
			> player clicks on action from menu (ATTACK, USE, DROP ITEM, etc)
				> action affects other visual elements: 
				   > ATTACK makes hand bounce once, enemy bounce once as it takes damage
				   > DROP ITEM removes image from HUD, places image in room
				   > USE LANTERN changes background image of current view (ONLY if room = dark)

	   > after action is complete (battle) item remains in hand unless HUD is clicked on again (as part of toggle)
